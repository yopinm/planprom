หน้า 6 / 8 · SECTION TYPES
🧱 Section Types 7 patterns ที่ใช้สร้างฟอร์มได้ทุกแบบ
1. Identity Section
ข้อมูลผู้กรอก — ใช้เกือบทุกฟอร์ม
Pre-built fields: คำนำหน้า · ชื่อ-นามสกุล · ตำแหน่ง · แผนก · รหัส · เบอร์โทร · อีเมล · ที่อยู่
JSON: { type: 'identity', preset: 'employee'|'customer'|'student' }
2. Fields Section
ฟิลด์อิสระ — admin กำหนดเอง
Configuration: array of fields · grid layout (1-4 cols)
JSON: { type: 'fields', cols: 3, fields: [{lbl, type, required}] }
3. Choices Section
checkbox/radio list · ใช้กับประเภท/เงื่อนไข
Modes: single (radio) · multi (checkbox) · yes-no
JSON: { type: 'choices', mode: 'single', options: ['ลาป่วย','ลากิจ',...] }
4. Table Section
ตาราง — ใช้กับ list items + calc
Features: dynamic rows · auto-calc (subtotal, VAT, total) · running number
JSON: { type: 'table', columns: [...], rowCount: 10, calc: {vat: 7} }
5. Paragraph Section
ข้อความคงที่ — closing, terms, instructions
Use case: "จึงเรียนมาเพื่อ..." · "ข้าพเจ้ายืนยันว่า..." · เงื่อนไข
JSON: { type: 'paragraph', text: '...', style: 'normal'|'italic'|'bold' }
6. Signature Section
ช่องลงนาม · 1-4 ฝ่าย
Roles: ผู้ยื่น · หัวหน้างาน · HR · ผู้รับเอกสาร · พยาน
JSON: { type: 'signature', signers: [{role, includeApproval}] }
7. Attachment List Section
รายการเอกสารแนบ · checklist + "อื่นๆ" line
Features: pre-defined items + free-text last option
JSON: { type: 'attachment-list', items: ['ใบรับรองแพทย์', 'สำเนาบัตร'], includeOther: true }

💡 Cumulative revenue: Form Engine ขายดีกว่า Planner เพราะ (1) ตลาดใหญ่กว่า — ฟอร์มไทยเป็นพันแบบ · (2) buy intent สูงกว่า — user ต้องการตอนนี้ ไม่ต้องวางแผน · (3) repeat purchase — บริษัทใช้หลายฟอร์ม
⚠️ Risk + ข้อระวัง:
1. Legal compliance — ใบกำกับภาษี/ใบเสร็จต้องตรงตาม spec กรมสรรพากร · ใบสำคัญรับเงินต้องมี VAT format ถูกต้อง
2. Trademark — ห้ามใช้ตราของหน่วยงาน (เช่น ตราครุฑ, ตราโรงเรียน) ใน template ขาย
3. Standard variations — บริษัทแต่ละที่มี header ของตัวเอง → template ต้องเป็น "blank skeleton" ไม่ใช่ "completed form"
4. Font + print quality — Form ส่งเข้าหน่วยงานราชการ → ต้อง print ได้ชัด · scan ได้ดี
5. Buddhist year — default พ.ศ. แต่ต้อง toggle ได้ (บางฟอร์มต้อง ค.ศ.)
